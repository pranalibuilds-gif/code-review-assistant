# Changelog

All notable changes to **CodeSage** will be documented in this file.

## [0.1.0] - Phase 12 - Documentation & Portfolio
- Completed project documentation.
- Generated Architecture, ER, and Pipeline diagrams.
- Created technical documentation for API and Database.
- Finalized root README.

## [0.0.10] - Phase 11 - Quality & Hardening
- Established Pytest suite for backend services.
- Implemented unit and integration tests.
- Hardened upload security (Zip Slip protection).
- Standardized API response format.

## [0.0.9] - Phase 10 - Administration
- Implemented System Health Service.
- Added background job registry.
- Implemented immutable Audit Logging.
- Built Admin Dashboard UI.

## [0.0.8] - Phase 8 - Dashboard & UI
- Built React-based Dashboard.
- Integrated Recharts for Quality Trends.
- Implemented identity and authentication frontend.

## [0.0.7] - Phase 7 - Canonical Aggregation
- Built the canonical report engine.
- Implemented weighted scoring (40/25/20/15).
- Implemented JSON report caching.

## [0.0.6] - Phase 6 - AI Review
- Integrated Ollama Local LLM.
- Implemented structured AI prompting.
- Added AI-based educational findings.

## [0.0.5] - Phase 5 - Static Analysis
- Integrated Pylint, Bandit, and Radon.
- Implemented parallel analyzer orchestrator.

## [0.0.4] - Phase 4 - Submission Pipeline
- Built the multi-source ingestion pipeline (Paste, Upload, GitHub).
- Implemented workspace isolation.

## [0.0.3] - Phase 2-3 - Identity & Data
- Established PostgreSQL and Alembic foundation.
- Implemented JWT authentication and RBAC.

## [0.0.1] - Phase 0-1 - Foundation
- Initial project skeleton and engineering standards.
- Repository initialization.
